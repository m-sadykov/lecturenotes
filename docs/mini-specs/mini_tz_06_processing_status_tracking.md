# Мини-ТЗ 06: Экран обработки и трекинг статусов

## Цель
Отображать пользователю прогресс обработки лекции и корректно обрабатывать закрытие экрана.

## Scope
- Экран `Processing` со стадиями: `Uploading`, `Transcribing`, `Generating notes`.
- Подписка на статус лекции в Firestore.
- Навигация после `ready` и поведение при `failed`.
- Возможность закрыть экран без остановки backend pipeline.

## Артефакты
- `ProcessingViewModel`.
- Маппинг backend-статусов в UI-состояния.
- Логика retry (если предусмотрена UX).

## Критерии приемки
- Статусы обновляются автоматически.
- Закрытие экрана не ломает обработку лекции.
- При `ready` данные доступны в detail экране.
