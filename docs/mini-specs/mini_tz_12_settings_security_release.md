# Мини-ТЗ 12: Settings, безопасность и готовность MVP

## Цель
Закрыть операционные и security-требования для релизной готовности MVP.

## Scope
- Экран `Settings`: language, delete audio after processing, restore purchases, support.
- Firestore/Storage security rules (доступ только владельцу `uid`).
- Гарантия, что AI API ключи находятся только на backend.
- Стабилизация и базовые unit-тесты ключевой логики.

## Артефакты
- `SettingsViewModel` и экран `Settings`.
- Набор security rules и проверка сценариев доступа.
- Unit-тесты на доменную и сервисную логику.
- Release checklist MVP.

## Критерии приемки
- Настройки сохраняются и применяются.
- Невладелец не может читать/писать чужие данные.
- Клиент не содержит секретов AI API.
- Чеклист MVP из ТЗ полностью закрыт.
